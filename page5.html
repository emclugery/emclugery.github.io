Projet 5 : Installation poste de développement


Pour ce projet d'installation de poste de développement, l'objectif est de développer un outil permettant de générer des PDF contenant les résultats trimestriels d'un client à destination de ses équipes commerciales régionales. Le but est de traiter et de formater les informations brutes fournies par le client pour les rendre lisibles et attrayantes. Le travail doit être réalisé de manière récurrente et accessible à tous les membres de l'équipe de développement. L'environnement de développement est Dockerisé, paramétrable et documenté, avec des conteneurs échangeant automatiquement leurs données via des volumes mappés. Les langages et outils utilisés comprennent PHP, Bash, les filtres Unix, ainsi que les images Docker spécifiques fournies (sae103-php, sae103-qrcode, sae103-imagick, sae103-html2pdf).


Partie 1 :


La première partie du projet comprend plusieurs tâches dont notamment la création du fichier de configuration "regions.conf" contenant toutes les informations nécessaires sur les régions, le traitement de fichier de données d'une région pour extraire trois fichiers nettoyés (texte.dat, tableau.dat et comm.dat), la création d'un document HTML avec des modèles de page et une feuille de style CSS, ainsi qu' une documentation détaillée et un fichier récapitulant les tâches réalisées et la répartition du travail dans le groupe.


Traitement du fichier de données d’une région pour en extraire 3 fichiers nettoyés prêts à être fusionnés avec le fichier HTML.

Création d'une page HTML et CSS regroupant tous les éléments nécessaires.


Deuxième partie :

Dans la deuxième partie de ce projet, nous devions tout d'abord, convertir les avatars fournis par le client en images PNG en nuances de gris, avec une taille de 200x200 pixels et en retirant le bandeau "Created on Face Co". Un script Bash a été développé pour automatiser ce processus, fonctionnant dans un conteneur basé sur l'image sae103-imagick. Ensuite, nous devions générer un QRCode pour la dernière page du document. Puis par la suite, nous avons transformé le modèle HTML précédent en un script PHP qui viendra remplacer les textes et parties factices par les données réelles extraites des fichiers préparés précédemment. Et l'étape final de ce projet consistait donc à générer les PDF correspondants aux régions grâce à un script.


Génération du QRCode de la dernière page de fichier


Transformation du modèle HTML créé en un script PHP qui va remplacer (fusionner) les textes et les parties factices par les vraies données issues des 3 fichiers .dat préparés
